# Conflict Resolution Rules

## Priority order
1. Foundational intent from `/seed`.
2. More specific and implementation-ready later canon may refine earlier broad wording.
3. Production-readiness constraints override prototype-style language.
4. Concrete testable requirements override aspirational prose.
5. If unresolved, preserve both + register conflict.
6. Never silently choose.

## Resolution protocol
- Record conflict in `04a_conflict_register.md`.
- Propose canonical interpretation with confidence.
- Attach ADR/decision when finalized.
- Keep unresolved items blocking if behavior-affecting.
