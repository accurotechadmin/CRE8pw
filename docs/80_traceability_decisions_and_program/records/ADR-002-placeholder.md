# ADR-002: Traceability Matrix Minimum Schema
- Status: accepted
- Date (UTC): 2026-04-29
- Owners: Architecture Governance WG
- Reviewers: Program Traceability WG, Docs Governance WG

## Context
The traceability matrix became a central artifact for proving requirement-to-verification coverage. Without a minimum schema, rows varied by session and impaired automation and audit consistency.

## Decision
CRE8 defines a minimum mandatory schema for every traceability row and binds it to merge acceptance for normative changes.

Normative decision requirements:
- **ADR-002-REQ-0001**: Every traceability row **MUST** include `Requirement ID`, `Source Document`, `Verification Hook ID`, `Verification Mode`, `Evidence Artifact`, and `Owner`.
- **ADR-002-REQ-0002**: `Verification Mode` **MUST** be `automated` or `manual`; manual rows **MUST** include an explicit automation target in backlog tracking.
- **ADR-002-REQ-0003**: Rows without a resolvable requirement backlink **MUST** fail `docs:ssot:sync-check`.
- **ADR-002-REQ-0004**: Acceptance bundles **MUST** include a current traceability report generated by `docs:ssot:report`.

## Alternatives considered
- Free-form matrix rows with reviewer judgment only (rejected: not machine-checkable).
- Per-domain custom schemas (rejected: excessive complexity for Phase 1).

## Consequences
- Positive:
  - Stable parser contract for lint and reporting automation.
  - Stronger audit evidence quality and reduced session-to-session interpretation drift.
- Negative:
  - Requires periodic schema migrations when adding new mandatory fields.
- Neutral:
  - Optional non-schema commentary remains allowed outside row fields.
