# Key Lifecycle And Cryptography Spec

## Purpose
Define deterministic lifecycle and cryptographic controls for CRE8 key material across issuance, proof validation, rotation, revocation, and propagation.

## Normative requirements
- **CRE8-SEC-REQ-0001**: Every minted principal **MUST** receive an ID keypair at mint time, and that ID keypair **MUST** be recorded as the lineage root for all descendant Utility keypairs.
- **CRE8-SEC-REQ-0002**: Utility keypairs **MUST** be context-scoped and **MUST NOT** be reused across unrelated applications, services, devices, or tenant workflows.
- **CRE8-SEC-REQ-0003**: Private key material generated by CRE8-managed workflows **MUST** be revealed exactly once at issuance; subsequent retrieval attempts **MUST** return deterministic deny outcomes defined by [`ERROR_CODE_CATALOG.md`](ERROR_CODE_CATALOG.md).
- **CRE8-SEC-REQ-0004**: Proof-based request verification **MUST** require key identifier, nonce, timestamp, and signature, and verification **MUST** reject requests failing any component before policy evaluation.
- **CRE8-SEC-REQ-0005**: Replay protection **MUST** enforce nonce uniqueness within the configured replay window and **MUST** enforce timestamp skew limits for all signed requests.
- **CRE8-SEC-REQ-0006**: Revocation and rotation transitions **MUST** be enforced immediately for all affected credentials, including descendant utility credentials when ancestor policy requires descendant impact.
- **CRE8-SEC-REQ-0007**: Lifecycle state transitions (`issued`, `active`, `suspended`, `rotated`, `revoked`, `expired`) **MUST** emit immutable governance/audit events with actor, scope, prior state, next state, and timestamp.
- **CRE8-SEC-REQ-0008**: Lifecycle writes that change credential usability **MUST** be transactionally committed together with corresponding audit events so that policy state and evidence cannot diverge.

## Verification hooks
- **HOOK-SEC-LIFECYCLE-PROPAGATION**: Execute lifecycle test suite proving immediate revoke/rotate effect on direct and descendant credentials.
- **HOOK-SEC-PROOF-VALIDATION**: Execute negative/positive proof verification scenarios for nonce, timestamp skew, signature mismatch, and missing proof fields.
- **HOOK-SEC-AUDIT-IMMUTABILITY**: Verify lifecycle transition writes always include immutable event records and fail closed on persistence mismatch.
