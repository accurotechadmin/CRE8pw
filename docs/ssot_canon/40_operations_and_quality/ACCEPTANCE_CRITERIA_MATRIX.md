# Acceptance Criteria Matrix (SSOT)

_Status: adopted_
_Last updated (UTC): 2026-04-28_

Canonical terminology: `docs/ssot_canon/10_product_and_architecture/CANONICAL_TERMINOLOGY.md`

## Purpose
Provide explicit, route-level Given/When/Then criteria (including negative and edge conditions) to reduce interpretation drift between backend, frontend, QA, and operations.

## Usage contract
- This matrix is normative for QA acceptance and release signoff.
- OpenAPI remains normative for request/response shape; this matrix is normative for behavioral intent.
- For conflicts, update both this matrix and OpenAPI-related artifacts in the same PR.

## Route acceptance matrix

| Capability | Route(s) | Positive acceptance (Given/When/Then) | Negative/edge acceptance | Evidence required |
|---|---|---|---|---|
| Service reachability | `GET /` | Given service is healthy, when route is requested, then success envelope is returned with `meta.envelope_version`. | If responder fails, return `500 internal_error` with `request_id`. | Contract test + health smoke output |
| Deep health | `GET /health` | Given dependencies are healthy, when requested, then all subsystem checks pass. | If any dependency fails, response indicates degraded/fail status and preserves correlation metadata. | Ops smoke + dashboard evidence |
| JWKS publishing | `GET /.well-known/jwks.json` | Given active signer set exists, when requested, then JWKS contains active verification keys. | During key rotation, overlapping key set is present until TTL drain completes. | Rotation rehearsal log + contract check |
| UI shell fallback | `GET /ui/{route}` | Given a valid SPA route request, when requested, then UI shell/static asset response is returned for client-side routing. | Unknown static assets return `404`; route fallback must not leak API-only security headers/cookies. | Browser parity check + security header verification |
| Owner signup | `POST /console/owners` | Given valid email/password + invite code, when submitted under default policy, then owner principal is created and success envelope returned. | Missing/invalid invite code emits `422 validation_failed` or `403 forbidden`; duplicate owner/email emits `409 conflict`; invalid payload emits `422 validation_failed`. | Contract tests for 201/403/409/422 + config-mode evidence for open-signup deployments |
| Owner login | `POST /api/auth/login` | Given valid owner credentials, when submitted, then owner JWT + refresh issued. | Invalid credentials -> `401 auth_invalid`; malformed payload -> `422`. | Auth contract + security tests |
| Key login | `POST /api/auth/key-login` | Given valid key credentials and policy compliance, when submitted, then key JWT + refresh issued. | Revoked/suspended/expired key or policy violation returns `401/403` as applicable; device claim/header mismatch on protected route returns `401 auth_invalid` (`token_device_mismatch`). | Contract + security regression suite |
| Refresh | `POST /api/auth/refresh` | Given valid refresh token family state, when submitted, then access token rotates and family state updates atomically. | Replay/invalid refresh returns `401 auth_invalid`; invalid body returns `422`. | Refresh replay security tests |
| Feed read | `GET /api/feed` | Given key JWT and required headers, when requested, then feed returns scoped content in stable order. | Missing/invalid auth -> `401`; disallowed scope -> `403`; limiter exceeded -> `429`. | Gateway contract + rate-limit tests |

| Post create | `POST /api/posts` | Given key has `posts:create` and allowed scope, when submitted, then post is created and visible by policy. | Use-key mutation restriction or missing permission -> `403`; invalid payload -> `422`. | Gateway contract + authz tests |
| Post read/edit | `GET/PATCH /api/posts/{postId}` | Given post visible to key, when requested/edited by authorized actor, then success envelope returned and revisions recorded. | Non-visible/non-existent -> `404`; unauthorized edit -> `403`; invalid edit payload -> `422`. | Contract + post revision verification |
| Post flag | `POST /api/posts/{postId}/flags` | Given visible post and valid reason, when submitted, then flag record is persisted. | Invalid reason/payload -> `422`; non-visible post -> `404` or policy outcome. | Contract + moderation audit checks |
| Comment list/create | `GET/POST /api/posts/{postId}/comments` | Given post visibility and comments policy, when requested/submitted, then comment list/create succeeds per permissions. | Disabled comments or missing permission -> `403`; invalid payload -> `422`. | Contract + policy mapping tests |
| Gateway content command-path authority | `POST /api/posts`, `PATCH /api/posts/{postId}`, `POST /api/posts/{postId}/flags`, `POST /api/posts/{postId}/comments` | Given gateway content mutation requests pass surface auth and PDP allow checks, when routed, then canonical content command handlers execute (`CreatePost`, `EditPost`, `FlagPost`, `CreateComment`) and envelope/detail-code semantics remain unchanged. | Any bypass of canonical content command handlers or mismatch in canonical `403/404/422` outcomes blocks release. | Command-handler integration report + contract regression output |
| Console posts | `GET/POST /console/api/posts` | Given owner JWT (+CSRF for writes), when requested, then Console BFF posts-governance orchestration executes and governance listing/create actions succeed. | Missing owner JWT -> `401`; invalid CSRF -> `403`; invalid payload -> `422`. | Console contract + CSRF tests |
| Keychain list/create | `GET/POST /console/api/keychains` | Given owner JWT, when requested, then keychain inventory/create succeeds with keychain principal semantics. | Invalid creation payload -> `422`; auth failure -> `401/403`. | Keychain contract tests |
| Keychain membership mutate | `GET /console/api/keychains/{keychainId}/members`, `POST /console/api/keychains/{keychainId}/members`, `DELETE /console/api/keychains/{keychainId}/members/{memberKeyId}` | Given owner JWT and valid member class, when mutating membership, then Console BFF keychain-governance orchestration executes and effective snapshot recomputation is triggered and auditable. | Duplicate member -> `409`; nested keychain attempt -> `422/403`; oversized membership (>50) blocked. | Keychain invariants tests + audit events |
| Keychain membership command-path authority | `POST /console/api/keychains/{keychainId}/members`, `DELETE /console/api/keychains/{keychainId}/members/{memberKeyId}` | Given console keychain membership mutation requests pass owner auth, CSRF, and PDP allow checks, when routed, then canonical keychain membership command handlers execute (`AddKeychainMember`, `RemoveKeychainMember`) and invariant/audit semantics remain unchanged. | Any bypass of canonical membership command handlers or invariant-denial mapping drift blocks release. | Keychain membership command integration report + invariant regression output |
| Keychain resolve | `GET /console/api/keychains/{keychainId}/resolve` | Given valid keychain and owner JWT, when requested, then effective permissions/scope plus lineage projection returned. | Missing keychain -> `404`; unauthorized -> `401/403`. | Contract + lineage tests |
| Console read query-path authority | `GET /console/api/posts`, `GET /console/api/keychains`, `GET /console/api/keychains/{keychainId}/members`, `GET /console/api/keychains/{keychainId}/resolve` | Given console read requests pass owner auth and PDP allow checks, when routed, then canonical query handlers execute (`ListConsolePosts`, `ListKeychains`, `GetKeychainMembers`, `ResolveKeychainEffective`) and response envelope/detail-code semantics remain unchanged. | Any bypass of canonical console query handlers or detail-code drift blocks release. | Console query-handler integration report + route parity output |
| BFF read-to-query integration parity | Migrated gateway + console read route families (`GET /api/feed`, `GET /api/posts/{postId}`, `GET /api/posts/{postId}/comments`, `GET /console/api/posts`, `GET /console/api/keychains`, `GET /console/api/keychains/{keychainId}/members`, `GET /console/api/keychains/{keychainId}/resolve`) | Given migrated read-route requests traverse surface BFF orchestration, when executed, then BFF modules dispatch reads through canonical `QueryBus` and query handlers and preserve canonical envelope/detail-code outcomes. | Any read-path bypass of `QueryBus`/canonical query handlers or latency/error semantic drift blocks release. | BFF read integration suite + route parity + latency report |
| BFF write-to-command integration parity | Migrated gateway + console write route families (`POST/PATCH /api/*`, `POST/DELETE /console/api/*` protected mutation routes) | Given migrated write-route requests traverse surface BFF orchestration and authorization/CSRF obligations, when executed, then BFF modules dispatch writes through canonical `CommandBus` + `TransactionalCommandExecutor` and preserve canonical envelope/detail-code outcomes. | Any write-path bypass of command handlers/transaction boundary or security/detail-code regression blocks release. | BFF write integration suite + contract/security regression report |
| Feed ordering projection freshness | `GET /api/feed` | Given committed post/comment/flag command events, when feed is requested, then feed ordering is served from `feed_ordering_projection` updated synchronously through canonical projector flow and returns deterministic ordering for the same actor/scope input. | Projection update failure rolls back command transaction and no stale projection write is exposed; duplicate source event does not duplicate projection rows. | Projection freshness suite + idempotency regression output |
| Keychain effective projection freshness | `GET /console/api/keychains/{keychainId}/resolve` | Given committed keychain membership or lifecycle command events, when resolve is requested, then effective permissions/scope and lineage summary are served from `keychain_effective_projection` updated synchronously through canonical projector flow. | Projection update failure blocks command success and no stale resolve projection response is exposed; duplicate source event produces deterministic replay no-op and does not duplicate projection receipts. | Keychain projection freshness suite + replay/idempotency regression output |
| Async projection worker reliability | CQRS-lite projection worker path (`ARCH_PROJECTION_ASYNC=true`) | Given async projection mode is enabled, when command/domain events are queued, then projector workers apply events with deterministic retry semantics and idempotent duplicate handling. | Terminal projector failures route events to dead-letter queue, emit canonical `projection.worker.dead_lettered` observability events, and do not violate command/event atomic durability guarantees. | Failure-injection suite + retry/DLQ evidence report |
| Async projection health visibility | `GET /health` with async projections enabled | Given async projection mode is enabled, when `/health` is requested, then response includes `projection_async` lag/depth subcheck fields and reports accurate freshness state. | Lag/depth threshold breach marks health status `degraded` and emits queue/alert events; malformed subcheck payload blocks release. | Health contract tests + async smoke output |
| Command failure and projection latency alerting | Operational dashboards/alerts (`command.*`, `projection.*`, `alert.*`) | Given stage/prod telemetry is active, when command failure rate or projection lag exceeds SLO thresholds, then dashboards surface breach and alert policies trigger on-call notification with request-correlation context. | Missing alert route, missing dashboard signal, or unresolved threshold breach blocks async projection activation outside local environments. | Alert drill evidence + dashboard snapshot pack |
| Invite issue | `POST /console/api/invites` | Given owner JWT and valid target constraints, when submitted, then Console BFF governance orchestration executes and invite receipt persisted with expiry. | Invalid expiry/target -> `422`; unauthorized -> `401/403`. | Console contract tests |
| Key issue/lifecycle | `POST /console/api/keys` + `POST /console/api/keys/{keyId}/lifecycle` | Given owner JWT and envelope bounds, when issuing/lifecycle actioning, then Console BFF governance orchestration executes and invariants and lineage are preserved atomically. | Over-scoped child or invalid transition -> `403/409/422`. | Security + contract lifecycle tests |
| Moderation actions | `POST /console/api/posts/{postId}/moderation`, `POST /console/api/posts/{postId}/comments/{commentId}/moderation` | Given owner/admin authority, when moderation action submitted, then Console BFF moderation orchestration executes and target state transition and audit event are written together. | Invalid transition -> `409/422`; unauthorized -> `403`. | Moderation integration tests |
| Gateway read caching parity | `GET /api/feed`, `GET /api/posts/{postId}/comments` | Given identical actor/scope/query inputs and warm cache entry, when requested, then response envelope/status/detail-code parity matches uncached execution. | Cross-principal replay attempt must miss cache; affected writes invalidate stale entries before subsequent reads. | Cache correctness + invalidation tests |
| Console inventory cache isolation | `GET /console/api/posts`, `GET /console/api/keychains`, `GET /console/api/keychains/{keychainId}/members` | Given owner JWT and warm short-TTL inventory cache, when requested, then response parity matches uncached execution and owner-scoped data remains consistent. | Different owner principal must not receive cached inventory from another owner; expired TTL forces refresh; cache adapter fault bypasses cache without failing route. | Console cache TTL/leakage tests |
| Console CSRF recovery hints | `POST /console/api/posts`, `POST /console/api/keychains`, `POST /console/api/invites`, `POST /console/api/keys`, `POST /console/api/keys/{keyId}/lifecycle` | Given canonical CSRF deny code, when request fails with `403 forbidden`, then Console BFF attaches deterministic diagnostics hint (`refresh_csrf`, `retry_write`, `re_auth_owner_session`) without changing canonical detail code. | Non-CSRF `403 forbidden` responses must not include CSRF diagnostics hints. | CSRF UX/security parity tests |
| Surface route-to-BFF integration parity | Migrated gateway and console route families (`/api/*` feed/posts/comments and `/console/api/*` posts/moderation/governance) | Given migrated route family requests, when routes execute under integration suite, then each request traverses canonical surface controller to matching surface BFF orchestration module and returns canonical envelope/detail-code semantics. | Any bypass to non-canonical orchestration path fails integration checks and blocks release. | Surface integration suite report by route family |
| Legacy orchestration dead-path removal | All migrated protected route families | Given runtime route handlers/controllers for migrated families, when static/dependency audit and integration negative-path checks run, then superseded non-BFF orchestration modules are absent from execution paths. | Any reachable superseded orchestration entrypoint or dependency reference fails dead-path audit and blocks release. | Diff-based dead-path audit + integration negative-path evidence |
| UI runtime error-state parity re-validation | Gateway + console runtime state mappings for migrated route families | Given canonical gateway and console API error outcomes (`401/403/404/409/422/429/500`), when UI parity regression suite executes, then UI runtime state transitions remain aligned with canonical status/detail-code semantics without introducing surface-specific code drift. | Any mismatch between canonical error outcomes and UI runtime state mappings blocks release. | UI parity regression report with per-route state mapping evidence |
| Resource-specific `404` semantics parity | Gateway and console resource routes plus unmatched-template routes | Given resource resolver misses and unmatched route templates, when UI and API parity suite executes, then resource resolver misses preserve canonical resource detail codes (`post_not_found`, `comment_not_found`, `key_not_found`, `keychain_not_found`) while unmatched-template misses preserve `route_not_found` semantics. | Any regression that collapses resource resolver and unmatched-template `404` semantics into a single UI/API behavior blocks release. | `404` parity regression bundle + route-level evidence matrix |
| CQRS-lite audit-first closure governance | Migrated protected route families across gateway and console surfaces | Given UC slices are complete, when closure review executes, then architecture/module/UI runtime/data model/observability/health/SLO/acceptance/traceability/ADR artifacts are synchronized with canonical CQRS-lite command/query/projection semantics. | Any cross-document inconsistency, missing traceability row, or missing ADR evidence blocks slice closure. | SSOT sync-check report + ADR review evidence + cross-doc diff package |

## Owner bootstrap policy mode
Default owner bootstrap mode is invite-gated. If open owner signup is enabled by configuration, release evidence must explicitly record that mode and associated abuse-case controls.

## Required negative-path baseline per route
At minimum for each route family, tests must cover: unauthorized, forbidden (where applicable), validation failure, and internal error envelope behavior.

## Manual UAT checklist linkage
Manual QA must execute route-family scenarios in `docs/ssot_canon/40_operations_and_quality/VERIFICATION_STRATEGY.md` and log request IDs for all failing paths.
