# Definition of Done

_Status: adopted_
_Last updated (UTC): 2026-04-09_

## Done means all are true
- Normative docs updated and internally consistent.
- Machine contracts updated for interface changes.
- Tests for positive/negative paths pass.
- Security controls and abuse-case expectations remain satisfied.
- Traceability matrix updated for changed capabilities.
- Release/readiness checks satisfied for impacted areas.
- Decision log and risk register updated when policy/architecture/risk profile changes.
- Evidence artifact attached using canonical templates.

## Required evidence checklist
- Verification commands and exit status.
- Contract deltas and compatibility classification.
- Startup/smoke evidence where operational behavior changed.
- Request IDs or correlation IDs for failures discovered during QA.
- Reviewer signoff notes.

## Not-done examples
- Doc-only claims without executable verification.
- Behavior changes lacking acceptance criteria updates.
- Policy changes without decision-table updates.
- Updated docs with stale metadata timestamps.
