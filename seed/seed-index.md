# CRE8 Seed Master Index

These seed documents are the source material used to create the rest of the canon SSOT guiding documents for the Credential Registry Engine (CRE8) software application platform. They establish architecture direction, invariants, governance constraints, and implementation-binding expectations that later SSOT artifacts must formalize and verify.

## Document List

- `seed-intro.md` — Canon introduction defining CRE8 identity, non-negotiable architecture invariants, and the dependency-bedrock policy.
- `CRE8_KEYPAIR_MODEL_BASE_INVENTORY.md` — Root architectural inventory for the ID-keypair-first model, authority chain, security posture, and acceptance baseline.
- `CRE8_PERMISSION_AND_DELEGATION_SEED.md` — Bounded delegation lattice and PDP-authoritative policy mutation rules.
- `CRE8_KEY_LIFECYCLE_AND_CRYPTOGRAPHY_SEED.md` — Lifecycle and cryptography requirements for ID/Utility keys, proof verification, and revocation behavior.
- `CRE8_SURFACES_AND_CLIENT_PARITY_SEED.md` — Surface model (owner/gateway/public) and UI/API parity guidance.
- `CRE8_CONTENT_AUDIENCE_AND_FEED_SEED.md` — Content targeting, audience semantics, comment permissions, and deterministic feed requirements.
- `CRE8_API_CONTRACT_AND_ERROR_SEED.md` — Envelope-first API contract, deterministic deny mapping, and protected-surface obligations.
- `CRE8_EXTENSIBILITY_AND_MODULE_PATTERN_SEED.md` — Module boundary and extension pattern constraints for safe platform growth.
- `CRE8_SEED_PRESERVATION_MATRIX.md` — Matrix of preserved truths, required redesigns, constrained assumptions, and follow-up authoring items.
- `CRE8_SEED_CANON_INDEX.md` — Reading order, domain ownership map, and governance constraints for the seed set.
- `CRE8_SEED_CANON_ASSESSMENT_REPORT.md` — Assessment of current seed maturity, risk register, and SSOT build-forward blueprint.
- `CRE8_REPO_STUDY_REPORT.md` — Consolidated repository-level analysis with cross-document findings and dependency-grounded directives.
