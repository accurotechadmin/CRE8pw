# Contribution Workflow (SSOT)

_Status: adopted_
_Last updated (UTC): 2026-04-08_

## Workflow
1. Identify impacted capability and SSOT docs.
2. Update contracts/policies/data docs first.
3. Update machine artifacts where applicable.
4. Update tests + verification docs + traceability.
5. Attach evidence templates and impact map.
6. Request required domain reviewers.

## Review policy
- No SSOT PR merges without explicit verification evidence.
- Breaking changes require architecture/security signoff.
- Unresolved gaps must be documented in known gaps tracker.
