# Definition of Done

_Status: adopted_
_Last updated (UTC): 2026-04-08_

## Done means all are true
- Normative docs updated and internally consistent.
- Machine contracts updated for interface changes.
- Tests for positive/negative paths pass.
- Traceability matrix updated.
- Release/readiness checks satisfied.
- Decision log and risk register updated if applicable.
- Evidence artifact attached.

## Not-done examples
- Doc-only claims without executable verification.
- Behavior changes lacking acceptance criteria updates.
- Policy changes without decision table updates.
