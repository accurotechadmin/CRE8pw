# CRE8 From-Scratch Reimplementation Section

_Status: draft_
_Last updated (UTC): 2026-04-08_

This root section is the dedicated workspace for rebuilding the CRE8 documentation system from the ground up using an **SSOT-first** method.

## Intent

1. Re-establish the documentation architecture beginning with a compact, authoritative SSOT core.
2. Build all other documentation layers around the SSOT core (implementation references, contributor docs, operational playbooks, onboarding guides).
3. Use the rebuilt documentation system to drive and constrain future CRE8.pw development.

## Initial deliverables in this section

- `PLAN_SSOT_FIRST_DOCUMENT_REBUILD.md` — staged plan for rebuilding documentation from scratch.
- `CORE_IDENTITY_AND_VALUE_PROPOSITION.md` — distilled ethos, concepts, utility, and differentiation of CRE8 based on existing repository documentation.
- `TECHNICAL_FOUNDATION_AND_BUILD_PLAN.md` — maturity analysis, current-state strengths/weaknesses, granular component mapping, and pristine technical build plan.

## Working principles

- **SSOT-first:** canonical docs define truth before implementation docs proliferate.
- **Contract-first:** routes, envelopes, auth surfaces, and security controls are specified before coding.
- **Policy-first:** security and authorization policy are treated as core product features.
- **Traceability:** every claim should map to contracts, code, and tests.
- **Layered docs:** foundational canon → technical references → workflow/ops guides.
